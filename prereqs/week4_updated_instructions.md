## 1. Prompting Requirements

Your prompt must explicitly ask the LLM to include the following details in its high-level cluster description.

### Required Description Content

| Requirement                     | Explanation                                                                                                                                        |
| ------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Components and Interactions** | Explain how the distinct parts of the cluster work together.                                                                                       |
| **Quality Attributes**          | Include the non-functional requirements achieved by the architecture, such as scalability, security, maintainability, reliability, or performance. |
| **Technology Used**             | Mention any frameworks, programming languages, tools, APIs, or technologies identified in the summaries.                                           |
| **Conciseness**                 | Keep each cluster description short and focused. The description must be **under 150 words**.                                                      |

### Prompting Tip

Update your prompt instructions so that they explicitly demand these data points. You should also enforce a strict word or sentence limit, for example:

> Keep the description under 150 words.

This helps ensure that the generated descriptions remain concise and consistent across clusters.

## 2. Clustering Algorithms

You are required to perform the hierarchical summarization process on clusters generated by **all three** of the following clustering algorithms:

1. **ARC**
2. **ACDC**
3. **LIMBO**

### Critical Pre-processing Step for ACDC and LIMBO

ACDC and LIMBO often produce clusters based on **inner classes**. However, the hierarchical summarization methodology requires **file-level clusters**.

Therefore, before running the LLM pipeline, you must convert the inner-class based clusters from **ACDC** and **LIMBO** into **file-based clusters** so that they match the structure produced by **ARC**.

## 4. Required Submission Format

For the hierarchical summarization results, your final submission must include a structured **CSV file per clustering algorithm**.

This means you should submit one CSV file for each of the following:

- `ARC`
- `ACDC`
- `LIMBO`

Each CSV file must contain **exactly** the following columns.

| Column Name   | Expected Content                                                                                          |
| ------------- | --------------------------------------------------------------------------------------------------------- |
| `cluster_ID`  | The unique identifier for the cluster.                                                                    |
| `files`       | A list of the files contained within that specific cluster.                                               |
| `title`       | The LLM-generated title for the cluster.                                                                  |
| `description` | The LLM-generated concise high-level summary, including components, quality attributes, and technologies. |

### Required CSV Structure

```csv
cluster_ID,files,title,description
```

### Example File Names

Recommended output file names:

```text
ARC_hierarchical_summarization_results.csv
ACDC_hierarchical_summarization_results.csv
LIMBO_hierarchical_summarization_results.csv
```
